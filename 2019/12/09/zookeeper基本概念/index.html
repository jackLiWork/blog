zookeeper基本概念

2019-12-09 评论

一、zookeeper节点中变量：

  1. epoch：为已经成功进行的选举轮次，每次leader变更，都会将epoch+1
  2. zxid组成：总共64位，高32位为epoch，低32位为发起的提案数count（即每发起一个提案就将count + 1）
  3. czxid：节点被创建时分配的递增序列号zxid
  4. mzxid：节点数据被修改时分配的递增序列号zxid
  5. pZxid：节点最新新增直系子节点的递增序列号zxid
  6. path：节点的路径，用于唯一标识一个节点
  7. version：节点数据的版本号，每调用一次setData更新一次，初始为0
  8. cversion：节点直系子节点被修改的次数（包括添加子节点和删除子节点操作），递增加1
  9. aversion：节点ACL被修改的次数，acl表示访问控制列表，用于验证节点的相关操作权限，初始为0
  10. ephemeralOwner：表示临时节点的会话Id
  11. datalenght：节点的数据长度，为所有子节点的数据长度加节点路径path长度
  12. numchildren：节点直系子节点的数量

二、创建节点：

  1. 入参：

    参数名 参数类型 说明
    path String 节点的完整路径（具有唯一性）
    data byte[] 节点的数据
    acl List 节点权限控制列表
    ephemeralOwner long 临时节点的会话Id，非临时节点会包含节点类型
    parentCVersion int 父母节点的CVersion
    zxid long 节点的初始事务ID，初始为1
    time long 节点创建时间
    outputStat Stat 会将创建节点的状态进行深度拷贝，供调用方使用

  2. 创建步骤：

    （1）根据入参初始化子节点的状态对象

    （2）获取节点的父节点，并根据path判重，此处会对父节点对象加锁

    （3）将父节点的CVersion加1，并设置pzxid为子节点的zxid

    （4）创建子节点，并加入父节点对应子节点集合中

    （5）将节点加入整个节点树节点集合中，并更新节点树的数据大小（包含创建节点数据大小和路径长度）

    （6）根据ephemeralOwner判断节点类型，并将其加入节点树的对应集合中

    （7）深度拷贝节点状态到outputStat入参中，并释放父节点对象锁

    （8）判断是否为quota节点和状态节点，并更新相关操作

    （9）更新此节点所在的quota节点树的状态，并进行相应容量和节点限制检查

    （10）更新新建节点的服务器监控指标

    （11）触发当前路径的节点创建事件监听器和父节点的子节点改变事件监听器

三、更新节点数据

  1. 入参：

    参数名 参数类型 说明
    path String 子节点的完整路径，具有唯一性
    data byte[] 需要给节点设置的数据
    version int 此次更新数据的节点版本号
    zxid long 当前leader下的对应事物Id
    time long 当前时间

  2. 更新步骤：

    （1）根据入参path获取当前节点对象

    （2）对当前节点对象进行加锁

    （3）更新当前节点的数据、mTime、mZxid、version属性

    （4）深度拷贝节点当前状态对象，用于更新操作完成时作为返回值

    （5）对当前节点对象释放锁

    （6）更新当前节点所在的quota节点树的状态，并校验相应约束

    （7）更新此节点的服务器监控指标

    （8）触发监听此节点数据改变事件的监听器

  3. 出参：返回此节点数据更新后深拷贝的状态对象

    四、删除节点

  4. 入参：

    参数名 参数类型 说明
    path String 子节点的完整路径，具有唯一性
    zxid long 当前leader下的对应事物Id

  5. 更新步骤：

    （1）根据入参path获取父节点对象

    （2）对父节点对象进行加锁

    （4）从父节点中移除此子节点

    （5）更新父节点的Pzxid属性，并释放锁

    （6）根据入参path获取当前节点对象

    （7）从节点集合中移除此节点对象

    （8）对当前节点对象加锁

    （9）更新节点树中记录的节点数据总大小

    （10）对当前节点对象释放锁

    （11）对父节点对象进行加锁

    （12）判断节点类型，并从对应类型节点集合中移除此节点

    （13）对父节点对象进行释放锁

    （14）若此节点是quota节点，从字段树中移除此节点路径

    （15）更新父节点对应的quota节点树状态，并校验相应约束

    （16）更新此节点的服务器监控指标

    （17）触发监听此节点数据改变事件的监听器

  6. 出参：无

五、获取节点数据

  1. 入参：

    参数名 参数类型 说明
    path String 子节点的完整路径，具有唯一性
    stat Stat 节点状态
    watch Watch 为此节点添加监听器

  2. 更新步骤：

    （1）根据入参path获取当前节点对象

    （2）对当前节点对象进行加锁

    （3）深度拷贝节点当前状态对象，用于查询完成时返回

    （4）向此节点的监听器集合中添加对应监听器

    （5）浅拷贝此节点当前当前数据，用于查询完成时返回

    （6）对当前节点对象释放锁

    （7）更新服务器监控的事件轨迹

  3. 出参：返回此节点的数据
