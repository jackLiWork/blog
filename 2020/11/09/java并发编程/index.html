java并发编程

2020-11-09 java学习 synchronized, 并发编程 评论

java并发编程

1、线程安全：多线程访问同一段代码，不会产生不确定的结果

2、优点：减少等待、容易共享资源、多核充分利用cpu

​ 缺点：访问冲突、锁竞争、死锁等

3、同步最小单位：cache line，大小为64字节

4、对齐方法：原子引用中追加15个变量，正好64个字节，实现一个引用在同一cache line

5、long/double不为原子操作

6、可见性：一个线程修改共享变量后，另一个线程立即可以读到修改后的值

7、有序性：程序执行的顺序按照代码的顺序执行，happens-before原则、synchronized、volatile

8、单线程下执行结果不能被改变：执行时重排、编译重排

9、new 对象：分配对象内存、初始化对象、赋值给引用，此步骤可能会被重排序，通过volatile可以防止涉及此变量的代码不被重排序，并且volatile修饰的变量写之后可以使得之前的共享变量（即使没有被volatile修饰）可见（jdk1.5之后），如果不加volatile，可能会造成变量未初始化完成前被使用

10、通过加版本号解决CAS的ABA问题

11、jdk1.6后synchronized会先自旋然后在入阻塞队列。synchronized代码块中调用notify后，需要代码块执行完后才会真正释放锁。synchronized代码块中抛异常的话，JVM会帮我们释放锁。

12、主要以redis、zookeeper、数据库（unique key）实现：正常unlock只能释放自己锁，超时可以释放他人的锁，redlock算法（在N个节点下，获取到>=N/2+1，才获取到锁）、主备

13、copyOnWriteArrayList（冗余，写时复制机制、适用读多写少情况，注意内存占用空间大的清咖滚，防止引发YGC或FGC）、ConcurrentHashMap

14、Array（基于数组）、Linked（基于列表，默认Integer.Max_VALUE）、SynchronousQueue(同步队列、没有容纳元素能力)

15、线程池ThreadPoolExecutor：coreSize足够大，满足平均吞吐量，当curPoolSize > coreSize&&阻塞对列已满&&curPoolSize < maxSize时，才会创建新threads，线程池中核心线程不会被回收，需要经过shutdown()操作

16、IO密集型：阻塞系数大。建议为2倍的CPU数。假设等待时间和运行时间一样，阻塞系数为0.5

最佳线程数=cpu线程数/（1-阻塞系数），阻塞系数=等待时间/（等待时间+运行时间）

17、fork/join思想

18、协程（轻量级线程、拥有独立的栈、局部变量和指令指针）

19、（1）双写，缓存设置不失效（数据不一致）、（2）缓存失效后只用一个线程去Db拿数据，需要同步、（3）缓存失效前先用一个请求线程去DB拿取，并更新缓存（流量特别高）

20、通过concurrentHashMap实现仅有一个线程去DB拿数据，通过putIfAbsent（）

21、异常、热点区域、在哪个线程、并行运行、锁粒度、并发修改变量可见行、
