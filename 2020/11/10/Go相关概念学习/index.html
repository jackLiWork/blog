Go相关概念学习

2020-11-10 Go学习 Go格式化, gofmt 评论

1、代码注释与c一样，块注释/* */，行注释//，最好使用标识符作为注释开头

  • 每个包都应该包含一段包注释（放置在package句之前），对于包含多个文件的包， 包注释只需出现在其中的任一文件中即可，为整个包的注释

  • 包注释会出现在 godoc 页面中的最上面

  • 在多段注释之间应以空行分隔加以区分

  • 注释无需进行额外的格式化，godoc 会自动将已缩进的文本以等宽字体显示，来适应对应的程序片段，注意注释在解析时都会被当成纯文本

2、命名规则

  • 文件名均由小写字母组成，若文件名由多个部分组成，则使用下划线 _ 对它们进行分隔
  • 包应当以小写的单个单词来命名，且不应使用下划线或驼峰记法
  • 包名应为其源码目录的基本名称
  • 有效的标识符必须以字符（可以使用任何 UTF-8 编码的字符或 _）开头，然后紧跟着 0 个或多个字符或 Unicode 数字
  • 变量（如owner）的get方法名采用首字母大写的变量名（如Owner），set方法名采用Set+首字母大写的变量名（如SetOwner）
  • 按照约定，只包含一个方法的接口应当以该方法的名称加上 - er 后缀来命名
  • Go 中约定采用驼峰命名来编写多个单词组成的标识符

3、Go 中分号

  • Go 的正式语法使用分号来结束语句，但是词法分析器会使用一条简单的规则来自动插入分号，因此对于满足下列规则的语句不用添加分号：

    • 若在新行前的最后一个标记为标识符（包括int和float64这类的单词）、数值或字符串常量之类的基本字面
    • 若在新行前的最后一个标记为以下标记之一
    • 无论如何，不应将一个控制结构（if、for、switch 或 select）的左大括号放在下一行

  • Go 程序只在诸如 for 循环子句这样的地方使用分号， 以此来将初始化器、条件及增量元素分开。如果在一行中写多个语句，也需要用分号隔开

4、Go 中下划线

  • 用在import：这种用法，会调用包中的init()函数，让导入的包进行初始化，但是却不使用包中其他功能。

  • 用在返回值：表示放弃对应位置的返回值

  • 用在变量：下划线用来判断结构体是否实现了接口（即实现了接口定义中所有函数）。如果没有实现，在编译的时候就能暴露出问题；否则，若代码中使用结构体没有实现的接口方法，编译器是不会报错的。

6、包的概念、导入和可见性

  • 每个 Go 文件都属于且仅属于一个包

  • 源文件中非注释的第一行用于指明这个文件的包名，如package 包名，当包名不为main时，编译后产生的对象文件将会是 包名.a

  • package main 表示一个可独立执行的程序，编译后为可执行文件，每个应用程序都包含一个或多个名为 main 的包

  • Go 的安装文件里包含了一些可以直接使用的包

  • 包的依赖关系决定了其构建顺序，每一段代码只会被编译一次，属于同一个包的源文件必须全部被一起编译，一个包即是编译时的一个单元

  • 包模型采用了显式依赖关系的机制来达到快速编译的目的，编译器会从后缀名为 .o 的对象文件（需要且只需要这个文件）中提取传递依赖类型的信息。

  • 你导入多个包时，最好按照字母顺序排列包名，导入语法：

    • 如果包名不是以 . 或 / 开头，如 “fmt” 或者 “container/list”，则 Go 会在全局文件进行查找；
    • 如果包名以 ./ 开头，则 Go 会在相对目录中查找；
    • 如果包名以 / 开头（在 Windows 下也可以这样使用），则会在系统的绝对路径中查找。
    • 如果你导入了一个包却没有使用它，则会在构建程序时引发错误

  • 包的可见性

    • 当标识符（包括常量、变量、类型、函数名、结构字段等等）以一个大写字母开头，如：Group1，这被称为导出（与java中的 public作用相同）
    • 标识符如果以小写字母开头，则对包外是不可见的，但是他们在整个包的内部是可见并且可用的（与java中的 private 作用相同）

6、go中的类型

  • 使用 var 声明的变量的值会自动初始化为该类型的零值
  • 类型可以是基本类型，如：int、float、bool、string
  • 类型可以是结构化的（复合的），如：struct、array、slice、map、channel，它使用 nil 作为默认值，Go 语言中不存在类型继承
  • 类型可以是只描述类型的行为，如：interface
  • 函数也可以是一个确定的类型，且函数也可以返回多个值(采用,分隔)
  • 使用 type 关键字可以定义你自己的类型

7、Go 程序的一般结构

  • 在完成包的 import 之后，开始对常量、变量和类型的定义或声明。
  • 如果存在 init 函数的话，则对该函数进行定义（这是一个特殊的函数，每个含有该函数的包都会首先执行这个函数）。
  • 如果当前包是 main 包，则定义 main 函数。然后定义其余的函数，首先是类型的方法，接着是按照 main 函数中先后调用的顺序来定义相关函数，如果有很多函数，则可以按照字母顺序来进行排序。

8、Go 程序在启动时，会按顺序导入所有被 main 包引用的其它包，然后在每个包中执行如下流程：

  • 如果该包又导入了其它的包，则从第一步开始递归执行，但是每个包只会被导入一次。
  • 然后以相反的顺序在每个包中初始化常量和变量，如果该包含有 init 函数的话，则调用该函数。
  • 在完成这一切之后，main 也执行同样的过程，最后调用 main 函数开始执行程序。

9、Go中的类型转换

  • Go 语言不存在隐式类型转换，所有的转换都必须显式说明
  • 当编译器捕捉到非法的类型转换（精度损失的转换等）时会引发编译时错误，否则将引发运行时错误
  • 具有相同底层类型的变量之间可以相互转换
