Go基本数据类型

2020-11-10 Go学习 Go基本数据类型 评论

1、常量

  • 使用关键字 const 定义，用于存储不会改变的数据，定义格式如下：

  • 常量类型只能是布尔型、数字型（整数型、浮点型和复数）和字符串型

  • 常量的值必须是能够在编译时就能够确定的（如计算表达式、内置函数等）

  • 数字型的常量是没有大小和符号的，并且可以使用任何精度而不会导致溢出，反斜杠 \ 可以在常量表达式中作为多行的连接符使用

  • 常量也允许使用并行赋值的形式

  • 每遇到一次 const 关键字，iota 就重置为 0

2、变量

  • 使用 var 关键字声明变量，定义格式如下：

  • 变量的命名规则遵循驼峰命名法，全局变量需要将首个单词的首字母大写

  • 函数体内声明的变量（包括参数和返回值变量）只作用于函数体内；代码块（用大括号括起来的部分）中声明的变量只作用于对应代码块中；处于内部作用范围内的同名变量会暂时隐藏外部作用范围的同名变量

  • 赋值给变量使用运算符等号=，需保证等号左右类型相同；声明新变量并直接初始化可以使用简短声明语法 :=

  • Go 是在编译时和运行时都可以进行变量类型的推断

  • go中所有的赋值都是值传递（注意此处将指针类型变量指向的内存中地址值传递也看做值传递），即使用等号 = 时会拷贝等号右边变量指向的内存中的值；若要获取一个对象的引用需要通过指针来操作，类似c中指针，但是go中指针不能参与运算，指针间的赋值操作仅仅是对对象引用的赋值。

3、init函数

  • 每个源文件都只能包含一个 init 函数。初始化总是以单线程执行，并且按照包的依赖顺序逆序执行
  • 不能够被人为调用，在每个包完成初始化后自动执行，并且执行优先级比 main 函数高
  • init 函数也经常被用在一个程序开始之前调用后台执行的 goroutine
  • init 函数可以用于全局变量的初始化，注意全局变量在函数体外不能单独赋值，但是可以声明时赋初始值

4、基本类型和运算符

  • Go 是强类型语言，不会进行隐式转换，任何不同类型之间的转换都必须显式说明

  • 只有两个类型相同的值才可以进行比较，注意接口类型值也可以参与比较

  • 变量格式化输出：

    • 使用 %t 来格式化输出布尔型变量
    • %d 用于格式化整数（%x 和 %X 用于格式化 16 进制表示的数字）
    • %g 用于格式化浮点型（%f 输出浮点数，%e 输出科学计数表示法）
    • %0d 用于规定输出定长的整数，其中开头的数字 0 是必须的。
    • %n.mg 用于表示数字 n 并精确到小数点后 m 位，除了使用 g 之外，还可以使用 e 或者 f，例如：使用格式化字符串 %5.2e 来输出 3.4 的结果为 3.40e+00
    • %v 来格式化输出复数
    • %b 是用于表示位的格式化标识符
    • %c 用于格式化输出字符，%v 或 %d 会输出用于表示该字符的整数，%U 输出格式为 U+hhhh 的字符串
    • 指针的格式化标识符为 %p

  • go原生支持复数，注意int、uint 和 uintptr类型长度与操作系统有关，浮点型仅有float32和float64（建议都使用float64），各个类型数据范围如下：

    • int8（-128 -> 127）

    • int16（-32768 -> 32767）

    • int32（-2,147,483,648 -> 2,147,483,647）

    • int64（-9,223,372,036,854,775,808 -> 9,223,372,036,854,775,807）

    • uint8（0 -> 255）

    • uint16（0 -> 65,535）

    • uint32（0 -> 4,294,967,295）

    • uint64（0 -> 18,446,744,073,709,551,615）

    • float32（+- 1e-45 -> +- 3.4 * 1e38），精确到小数点后7位

    • float64（+- 5 1e-324 -> 107 1e308），float64 精确到小数点后 15 位

    • complex64：32 位实数和虚数

    • complex128：64 位实数和虚数

  • 运算符优先级

    优先级 运算符

    7              ^ !
6              * / % << >> & &^
5              + - | ^

    4 == != < <= >= >
    3 <-
    2 &&
    1 ||

  • 类型别名，通过type关键字，如type TZ int

  • go中字符类型

    • byte 类型是 uint8 的别名，只占用 1 个字节

    • 两个字节采用int16 或者 int 类型来表示，表示采用前缀\u+长度为 4 的 16 进制数

    • 4 字节采用int类型来表示，表示采用前缀\U+长度为 8 的 16 进制数

4、字符串类型

  • go中字符串是 UTF-8 字符的一个序列，当字符为 ASCII 码时则占用 1 个字节，其它字符根据需要占用 2-4 个字节

  • go中字符串是一种值类型，且值不可变，字符串底层是字节的定长数组

  • Go 中的字符串是根据长度限定，通过len()可以获取字符串的长度

  • 注意go中对字符串使用比较运算符（==、!=、<、<=、>=、>）是通过在内存中按字节比较来实现字符串的对比

  • 当字符串中所有字符都为纯 ASCII 码时，可以通过中括号 [index]来获取，索引从0开始

  • 两个字符串可以通过+号进行拼接；若存在换行，加号必须在行尾（由于编译器行尾自动补全分号的缘故）

  • 任何类型转换为字符串总是成功的

4、时间和日期

  • Duration 类型表示两个连续时刻所相差的纳秒数，类型为 int64

  • 根据格式化字符串（注意格式化字符串中时间点必须是2006-01-02 xx:xx:xx.000000000）来将一个时间 t 转换为相应格式的字符串

    格式 含义
    01、 1、Jan、January 月
    02、 2、_2 日，_2表示如果日期是只有一个数字，则表示出来的日期前面用个空格占位。
    03、 3、15 时
    04、4 分
    05、5 秒
    2006、06、6 年
    -070000、 -07:00:00、 -0700、 -07:00、 -07、Z070000、Z07:00:00、 Z0700、Z07:00 时区
    PM、pm 上下午
    Mon、Monday 星期
    MST 美国时间，如果机器设置的是中国时间则表示为UTC
    .999999 9个数为返回秒小数点后多少位，最多到微秒
    .000000000 0个数为返回秒小数点后多少位，最多到纳秒

4、指针

  • Go 语言为程序员提供了控制数据结构的指针的能力，取地址符是 &，通过在指针类型前面加上 * 号来获取指针所指向的内容
  • 一个指针变量可以指向任何一个值的内存地址， 在 32 位机器上占用 4 个字节，在 64 位机器上占用 8 个字节
  • 当一个指针被定义但没有进行初始化时，它的值为 nil
  • 不能得到一个文本或const常量的地址
  • Go 语言指针运算是不被允许的，从而保证了内存安全
  • 指针也可以指向另一个指针，并且可以进行任意深度的嵌套，但是不建议这样做
  • 对一个空指针的反向引用是不合法的，并且会使程序崩溃
