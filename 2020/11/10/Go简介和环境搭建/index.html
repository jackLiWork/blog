Go简介和环境搭建

2020-11-10 Go学习 Go简介和环境搭建 评论

1、Go语言诞生

Go 语言起源 2007 年，并于 2009 年正式对外发布。2009 年 11 月 10 日，开发团队将 Go 语言项目以 BSD-style 授权（完全开源）正式公布了 Linux 和 Mac OS X 平台上的版本。同年 11 月 22 日公布了 Windows 版本。

2、Go 应用场景和特性

  • Go 语言与 C++ 类似，属于系统编程语言。
  • Go 语言设计的初衷主要应用于集群 Web 服务器。Go 语言对于高性能分布式系统领域有着更高的开发效率，并提供了海量并行的支持。
  • Go 语言是一门类型安全和内存安全的编程语言，Go 语言中仍有指针的存在，但并不允许进行指针运算
  • Go 语言对于网络通信、并发和并行编程有极佳支持，从而更好地利用大量的分布式和多核的计算机。Go 语言通过引入 goroutine 轻量级线程，然后通过 channel 来实现各个 goroutine 之间的通信，从而实现了分段栈增长和 goroutine 在线程基础上多路复用技术的自动化。
  • Go 语言没有类和继承的概念，但是它通过接口（interface）的概念来实现多态性

3、Go 语言环境安装：Mac 开发环境

  • 首先访问 https://golang.google.cn/dl/ ，点击 Apple macOS 中的链接下载软件包至本地。

  • 然后运行软件包，遵照提示点next按钮。

  • 安装程序将安装 Go 编译器、工具、库文件至 /usr/local/go 目录，并把 GOPATH=/usr/local/go/bin 目录添加至 PATH 环境变量中。

    • GOROOT就是go的安装路径
    • GOPATH作为import包时的搜索路径 ，其主要包含三个目录: bin、pkg、src，bin目录主要存放`go install编译可执行文件（多个项目可共用）， pkg目录存储预编译目标文件的地方，以加速程序的后续编译（主要是*.a文件，可以手动删除），src目录下主要存放go的源文件。

5、go get命令：go get 可以下载指定的软件包及其依赖

6、go build 和 go install命令：在没有附加参数的情况下运行时都将编译当前目录下的包

  • go build :
    • 如果包是 package main，那么 go install 将把它放在 $GOPATH/bin （如果有多个参数将使用 $GOPATH 的第一个参数）
    • 如果包不是 main 包，go install 将会编译包，并安装在 $GOPATH/pkg 中（包括依赖项）
  • go install：
    • 如果包是 package main，那么 go build 将把生成的可执行文件放在当前目录中

7、格式化命令：gofmt -w或go fmt -w

  • 优先使用tab制表符缩进

  • gofmt –w program.go 会对源文件进行格式化并覆盖原始内容（如果不加参数 -w 则只会打印格式化后的结果而不重写文件）

  • gofmt -w *.go 会格式化并重写所有 Go 源文件

  • gofmt dir 会格式化并重写 dir 目录及其子目录下的所有 Go 源文件
