QuorumCnxManager类详解

2020-01-21 评论

1、QuorumCnxManager概述

（1）作为leader选举时用于服务器间网络连接的管理器，采用tcp协议，包括网络层协议的初始化，通过阻塞队列实现发送和接收的异步处理，发送和接收队列中存储对象类型为ByteBuffer，并且是按接收顺序组成FIFO队列，分包是通过设置每个包前四个字节为包的长度来实现（依赖tcp协议来保证分包的正确性和顺序性）。

（2）采用tie-breaking机制（只接收sid大于自己sid的网络连接请求，否则断开此连接并由自己向对方发起新连接请求）解决两台服务器并发建立连接时产生重复网络连接

（3）包含一个信息发送队列，当连接的某一服务器挂了，发送给此服务器的消息将被放入此发送队列

4、QuorumCnxManager变量详解：

属性名 属性类型 属性说明
RECV_CAPACITY int 接收线程队列的最大容量，默认100
SEND_CAPACITY int 发送线程队列的最大容量，默认1，防止发送旧的notifications
PACKETMAXSIZE int 接收包的最大字节数，默认512K
observerCounter AtomicLong 观察者的数量
maxBuffer int 从网络读取数据时最大缓存大小，默认2048字节
cnxTO int tcp建立网络连接时的超时时间
self QuorumPeer 服务器管理类
mySid long 服务器对应sid
socketTimeout int 设置网络socket接收数据时等待的超时时间
view Map<Long, QuorumPeer.QuorumServer> 用于保存此服务器所在集群服务器中sid和对应服务器使用网络地址（ip和port）间的映射关系
listenOnAllIPs boolean 是否监听此服务器下的所有网卡（一个网卡对应一个ip地址），即创建socket时仅设置port，监听时会扫描此服务器下所有IP地址对应的port
ThreadPoolExecutor connectionExecutor 服务器间选举时，用于异步建立连接或异步接收连接时的线程池executor
inprogressConnections Set 用于异步初始化网络连接应用层协议时，保存连接到的服务器的sid
authServer QuorumAuthServer 用于此服务在接收连接请求后，提供鉴权检查
authLearner QuorumAuthLearner 用于此服务在发起连接请求后，提供鉴权检查
quorumSaslAuthEnabled boolean 是否异步接收连接请求或异步发起连接请求
connectionThreadCnt AtomicInteger 记录异步网络连接（包括接收和发起连接请求）时使用线程数量
senderWorkerMap ConcurrentHashMap<Long, SendWorker> 保存服务器sid与发送线程映射关系，此线程用于执行向此服务器发送消息的任务
queueSendMap ConcurrentHashMap<Long, ArrayBlockingQueue> 保存服务器sid与发送队列间映射关系，此队列用于保存待发送到此服务器的数据
lastMessageSent ConcurrentHashMap<Long, ByteBuffer> 保存服务器sid与最后发送数据间映射关系，此数据为已经发送给此服务器的最新数据
recvQueue ArrayBlockingQueue 此服务器的接收同步队列
listener Listener 监听器线程
threadCnt AtomicInteger 记录此服务器用于接收消息和发送消息时创建的线程数
tcpKeepAlive boolean 建立tcp连接时是否设置KeepAlive

6、receiveConnection()方法的详细处理流程：可以看到接收方仅接受发起方sid（发送方发起请求后会发送自己的sid等信息）比接收方sid大的连接请求，同时接收方会主动向被拒绝的发送方（sid比接收方sid小）发起连接请求并发送协议版本号和自己sid等信息。当接收方法接受发送方连接请求后，就会建立发送方的sid对应的接收处理线程和接收同步队列（当有消息到来时，进行接收并放入队列中，否则一直阻塞等待消息）、发送处理线程和发送同步队列（当发送同步队列有值时，进行网络发送，否则阻塞）。

![image-20200305220814282](/Users/lixiaolong/Library/Application Support/typora-user-images/image-20200305220814282.png)

7、QuorumCnxManager通过暴漏的toSend(Long sid, ByteBuffer b)方法主动发起网络连接流程：先将消息存入同步队列中，当发送线程容器中存在sid对应的线程，则表示此服务器已经与sid服务器建立连接，否则向sid的服务器发起连接请求。

![image-20200305234231661](/Users/lixiaolong/Library/Application Support/typora-user-images/image-20200305234231661.png)

8、主动发起请求的详细流程：此方法用于服务器去主动发发起建立连接，建立socket连接后会发起方会发送自己的sid等信息，当接收方接收到此消息后，会解析出发起方发送的sid等信息，并与自己的sid进行比较，当发现发起方的sid小于接收方时，接收方会发起关闭socket连接，然后由接收方重新发起socket连接，并发送自己sid等信息；如果发起方的sid大于接收方，则接收方会接受连接并建立发送方sid对应的发送线程和发送同步队列、接收线程和接收同步队列。

（1）此方法主要用于建立socket连接前一些准备工作，比如获取并更新sid对应的ip地址等
