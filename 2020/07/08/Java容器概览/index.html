java容器概览

2020-07-08 java容器学习 容器, 迭代器 评论
  1. Java容器概念：用来存某一类或具有继承关系的类对象，并提供添加、删除、查找等操作的类，就叫做容器或者集合。
  2. 容器的使用场景：在程序运行过程中，动态创建许多相同或具有继承关系的类对象，并且对象的数量根据运行情况动态变化；建立一类对象与另一类对象之间映射关系；提供某些操作以便我们可以遍历这些对象或快速定位到特定的对象。如Spring容器内部数据结构用到Map容器
  3. java容器基本分类：List 、Set、Map。

  • Set容器特点：容器中不允许出现重复的元素

  • List容器特点：容器中区分元素的顺序，通过序号存取相应位置的元素，且允许包含重复的元素

  • Map映射：Map是一种把键对象和值对象进行映射的集合，集合中映射关系需要满足一对一，即不能包含重复的键值且每个键最多只能映射一个值

  1. java容器总体接口继承图（面向接口编程，使用泛型）

  2. Collection接口：提供集合的通用操作方法

    （1）removeAll和containsAll以及retainAll方法的形参集合中泛型为通配符?、remove和contains方法的形参类型为Object原因：

    • 如果指定形参类型为范型参数类型，那么如果使用含有通配符这样的变量，在调用其方法时，无论传任何类型的对象，除非传null，否则编译器都会报错

    • contains方法依赖Object.equals()方法，而Object.equals()方法接收的参数类型正是Object类型，这也允许使用不同于集合中类的对象对集合中的元素进行检索

（2）addAll方法的形参集合中泛型为上限通配符、add方法的形参类型为泛型而不为Object的原因：

  • 如果形参类型是Object类型，集合中就可以插入不同的类型的元素，这样集合就被破坏，而当形参类型为泛型E时，就可保证插入的类型也为E，否则编译出错

  1. 迭代器模式作用：对不同容器对象中的各个元素提供统一的访问方式，而又不暴露这些容器的内部实现细节

  • 迭代器模式UML：

  • 实现Iterable接口允许对象成为Foreach语句的目标，就可以通过foreach语句来遍历你的底层序列

  • 在迭代的过程中（包括显式迭代和使用foreach迭代），s如果对迭代的容器进行了改变其容器大小的操作（不包括当前迭代器的remove操作），那么Java就会给出异常（如抛出ConcurrentModificationException异常），造成并发不一致问题。实现机制：内部维护了两个版本号，modCount是容器中的版本号，每次修改(增、删)容器都会加1；expectedModCount是迭代器中的版本号，在迭代器实例化时或者通过迭代器自身的remove操作时初始化为modCount，每次迭代都会检查这两个版本号是否一致。
