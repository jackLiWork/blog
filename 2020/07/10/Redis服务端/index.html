Redis服务端

2020-07-10 Redis学习 Redis, Redis服务端 评论

服务器的启动过程

服务器启动可以分为几部分：初始化服务器默认配置、解析配置文件并加载配置文件、执行服务器的初始化过程、通过持久化文件还原数据库、执行事件主循环，以下为具体的流程：

1、初始化服务器默认配置（执行initServerConfig函数）

  • 首先创建一个struct类型的redisServer变量server，用于维护服务器的状态

  • 初始化server的某些属性，如运行Id、默认运行频率、配置文件路径、运行架构、默认端口号、默认持久化条件、初始化服务器LRU时钟、创建命令表等

2、载入启动配置参数

3、初始化服务的数据结构（执行initServer函数）

  • 创建并初始化redisServer中重要的数据结构，如clients链表，db数据库数组、订阅相关属性、慢查询日志属性等

  • 为服务器设置进程信号处理器

  • 创建事件循环对象

  • 创建共享对象

  • 打开服务器的客户端连接监控端口并创建文件事件，绑定连接处理函数acceptTcpHandler

  • 创建时间事件，并为时间事件绑定时间事件处理函数serverCron

  • 创建或打开AOF文件，为AOF写入做好准备

  • 初始化系统后台线程，执行AOF持久化相关任务（如磁盘同步、文件资源释放和删除）Redis BIO详解

4、还原数据库状态

  • 如果服务器启用了AOF持久化功能，则载入AOF文件来还原数据库状态
  • 如果没有启用AOF持久化功能，则载入RDB文件来还原数据库状态
  • 打印载入文件并还原数据库所耗费的时长

5、执行主事件循环（详见Redis事件）

服务端维护客户端状态

1、客户端状态结构

一个服务器可以与多个客户端建立网络连接，服务器状态结构的clients属性保存了所有与服务器连接的客户端的状态结构，对应的struct结构对象redisClient如下：

flags含义如下：

标志 含义
REDIS_MASTER 标识客户端为master服务器
REDIS_SLAVE 标识客户端为slave服务器
REDIS_PRE_PSYNC 表示不能对此客户端标识的从服务器执行SYNC命令
REDIS_LUA_CLIENT 表示客户端是专门用于处理Lua脚本的伪客户端
REDIS_MONITOR 表示客户端正在执行MONITOR命令，即Redis服务器会向它发送接收到的命令
REDIS_UNIX_SOCKET 表示客户端与服务器通过使用UNIX套接字进行连接
REDIS_BLOCKED 表示客户端当前正处于阻塞状态，如被BRPOP或BLPOP命令阻塞
REDIS_UNBLOCKED 表示客户端当前已经从REDIS_BLOCKED标志的阻塞状态中被唤醒
REDIS_MULTI 表示客户端正在执行事务
REDIS_DIRTY_CAS 表示事务使用WATCH命令监视的数据库键已经被修改
REDIS_DIRTY_EXEC 表示事务在命令入队时出错，表示事务的安全性已经被破坏
REDIS_CLOSE_ASAP 表示客户端的输出缓冲区大小超出服务器允许范围，客户端会在时间事件中关闭
REDIS_CLOSE_AFTER_REPLY 表示由用户对此客户端执行了kill命令或者客户端发送的命令请求协议格式有误
REDIS_ASKING 表示客户端向集群节点发送了ASKING命令
REDIS_FORCE_AOF 表示客户端强制服务器将当前执行的命令写入AOF文件中，如PUBSUB命令和SCRIPT LOAD
REDIS_FORCE_REPL 表示客户端强制主服务器将当前执行的命令同步给所有从服务器，如SCRIPT LOAD
REDIS_MASTER_FORCE_REPLY 主从服务器进行命令传播期间，当主服务器的此标志位被设置时，从服务器才能向主服务器发送REPLICATION ACK命令，

2、创建普通客户端

客户端在使用connect函数连接到服务器时，服务端会为客户端创建相应的客户端状态结构redisClient，并将这个新的客户端状态结构添加到服务器状态结构clients链表的末尾

3、Lua脚本的伪客户端

服务器在初始化时就会创建一个伪客户端，去负责执行Lua脚本中包含的Redis命令，这个伪客户端在服务器状态结构的lua_client属性中维护。lua_client伪客户端在服务器运行的整个生命周期会一直存在，只有服务器被关闭时，这个客户端才会被关闭

4、AOF文件的伪客户端

服务器在载入AOF文件时会创建临时伪客户端，用于执行AOF文件中包含的redis命令，当AOF载入完成后，此伪客户端会被关闭。

5、输出缓冲区限制

  • 硬性限制：当缓冲区大小超出硬性限制大小之后，服务器马上关闭客户端。

  • 软性限制：当缓冲区大小超出软性限制大小但没有超出硬性限制，服务器通过obuf_soft_limit_reached_time属性记录客户端开始软性限制的时间，之后服务器会持续监视客户端，如果持续处于软性限制的时间超过服务器设定的时长，那么服务器将关闭客户端；相反地如果输出缓冲区在指定时长内回到正常状态，客户端将不会被关闭，并且obuf_soft_limit_reached_time属性值也会被清零。

6、关闭普通客户端：关闭客户端时会释放服务器端维护的客户端相关内存资源和句柄资源，关闭条件如下：

  • 客户端进程退出或被杀死
  • 客户端向服务器发送了带有不符合协议格式的命令请求
  • 客户端成为了client kill命令的目标
  • 用户为服务器设置了timeout配置选项，当客户端的空转时间超过timeout选项设置的值时
  • 客户端发送的命令请求的大小超过了输入缓冲区的限制大小
  • 要发送给客户端的命令回复的大小超过了输出缓冲区的限制大小
