Redis事件

2020-07-13 Redis学习 Redis, Redis事件 评论

Redis事件简介

redis服务器是一个单线程的事件驱动程序，处理的事件主要分为两类：文件事件和时间事件。文件事件是服务器对套接字操作过程的抽象，服务器通过监听这些套接字产生的网络事件来完成建立连接、读取命令等操作。时间事件是服务器对定期或周期操作过程的抽象，用于执行某些需要定期或周期执行的任务。但是注意，Redis主流程始终是通过单线程来轮流执行文件事件和时间事件的。

Redis事件初始化

Redis 服务器启动时，会调用 initServer 方法，首先会建立 Redis 自己的事件机制 eventLoop，然后在其上注册周期时间事件处理器，最后在所监听的 socket 上创建文件事件处理器，监听 socket 建立连接的事件

文件事件

Redis基于Reactor设计模式开发的网络事件处理器，底层采用I/O多路复用技术的LT模式来同时监听多个套接字，并为对应的事件分发相应的事件处理器。以下为采用epoll模式，监听文件事件并获取就绪文件事件的过程：

可以看到网络事件在epoll中类型被分为4类，分别是EPOLLIN、EPOLLOUT、EPOLLERR、EPOLLHUP，其含义如下

事件类型 事件发生含义
EPOLLIN 表示对应的文件描述符可以读（包括对端正常关闭连接；新连接请求；有数据到来）
EPOLLOUT 表示对应的文件描述符可以写（包括缓冲区满–>缓冲区非满；读写都被监听，当有读事件发生；新连接请求成功后）
EPOLLERR 表示对应的文件描述符发生错误，只有服务器采取读写等行为时，才会触发
EPOLLHUP 表示对应的文件描述符被挂断

文件事件处理器，目前Redis提供多种文件事件处理器，如连接应答处理器（acceptTcpHandler）、命令请求处理器（readQueryFromClient）、命令回复处理器（sendReplyToClient）等，不同的时间类型需要绑定不同的时间处理函数，即将文件事件对应aeFileEvent对象的rfileProc或wfileProc属性赋值为指向对应事件处理函数的指针。

  1. 连接应答处理器（acceptTcpHandler）：在Redis服务器进行初始化时候，程序会给服务器用于监听网络连接的文件事件绑定连接应答处理器，即将此事件表示的aeFileEvent对象的rfileProc属性赋值为指向连接应答处理器acceptTcpHandler的指针。当有客户端连接服务器时候，此文件事件就会产生AE_READABLE事件，然后调用acceptTcpHandler函数进行处理，并执行相应的应答操作。acceptTcpHandler函数具体执行过程如下：

时间事件

时间可以分为两类：定时事件和周期性事件，定时事件表示此事件仅被程序执行一次；周期性事件表示此事件会每隔一段事件让程序执行一次。以下是周期事件的执行流程：

目前Redis中常用的时间事件处理器为serverCron函数（默认100ms运行一次），用于定期检查服务器自身的资源和状态，确保服务器可以长期、稳定的运行，具体的工作包括：

  • 更新服务器的各类统计信息，如运行时间、占用内存、数据使用情况等
  • 清理数据库中过期键值对
  • 关闭和清理连接实效的客户端
  • 尝试进行AOF和RDB持久化操作
  • 定期触发向从服务器的同步操作
  • 定期与集群中其他的机器进行连接测试
