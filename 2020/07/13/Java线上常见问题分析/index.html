Java线上常见问题分析

2020-07-13 java学习 Java, 线上常见问题 评论

常见问题分类

java线上常见问题，目前按表现大致可以分为两方面：服务器异常和业务逻辑异常等。业务逻辑异常由于与业务具有强耦合性，需要结合具体逻辑才能进行最终的查找定位。因此本文主要讲解服务器异常造成的问题定位分析方法。服务器异常可以再细分为4类，分别是cpu使用率高、内存溢出、磁盘打满和网络。

cpu异常

cpu主要有两个指标cpu负载和cpu使用率。cpu使用率指的是程序在运行期间实时占用的cpu百分比。而cpu负载指的是一段时间内正在使用和等待使用cpu（即不可中断阻塞状态：一般表示进程正在跟硬件交互，并且交互过程中不允许被其他进程中断或打断）的平均任务数，也就是说cpu负载越高，表示cpu必须要在不同的工作之间进行频繁的切换。cpu利用率高，并不意味着负载就一定大。判断cpu出现问题的阈值：cpu使用率长时间>60%或者cpu负载>cpu核心*0.7，触发cpu使用率高的原因有很多，除了程序本身是计算密集型，程序中死循环、死锁、fgc都会触发cpu使用率高。
可中断阻塞状态：表示进程正在等待某个事件而被系统挂Q，当进程等待的事件发生时会进入R状态

cpu使用率高：

  1. ps -ef | grep java，获取Java进程的PID（注意使用jps命令可能会查不到运行的java进程）

    字段名 含义 字段名 含义
    UID 用户Id或用户名 STIME 系统启动时间
    PID 当前进程的PID TTY 登入者的终端位置
    PPID 当前父进程的PID TIME 当前进程已使用cpu的时长
    C CPU使用的资源百分比 CMD 启动当前进程使用的命令及参数

  2. top -p pid命令，查看某个PID进程使用情况

    第一行表示任务队列信息：(18:10:52)为系统当前时间，(up 9 days, 20:26)为系统运行时间、(1 user)当前登录用户数、(load average: 18.27, 18.92, 18.62)表示当前系统任务队列的平均长度，分别表示前1分钟、5分钟、15分钟的平均值

    第二行表示进程信息：(1 total)为进程总数，(0 running)为正在运行的进程数，(1 sleeping)为休眠的进程数，(0 stopped)停止的进程数，(0 zombie)表示僵尸进程数

    第三行表示CPU信息：(0.9%us)进程处于用户态占用CPU百分比，(0.4%sy)进程处于内核空间占用CPU百分比，(0.0%ni)用户进程空间内改变过优先级的进程占用CPU百分比，(98.5%id)空闲CPU百分比，(0.0%wa)等待输入输出的CPU时间百分比，(0.0%hi)硬件CPU中断占用百分比，(0.1%si)软中断占用百分比，(0.0%st)虚拟机占用百分比

    第四行表示内存信息：(8388608k total)物理内存总量，(7867360k used)使用的物理内存总量，(521248k free)空闲内存总量，(0k buffers)用作内核缓存的内存量

    第五行表示交换区信息：(2097152k total)交换区总量，(0k used)使用的交换区总量，(2097152k free)空闲交换区总量，(1213120k cached)已缓冲的交换区总量

    字段名 含义 字段名 含义
    PID 进程Id SHR 共享内存大小
    USER 进程所有者的用户名 S 进程的状态（S睡眠，R运行，Z僵尸进程、T调试）
    PR 进程优先级 %CPU 进程占用CPU的百分比
    NI nice值，复制表示高优先级，正值表示低优先级 %MEM 进程占用的物理内存百分比
    VIRT 进程使用的虚拟内存总量 TIME+ 进程总体使用cpu的时长
    RES 进程使用实际物理内存大小 COMMAND 打开此进程的命令

  3. ps -mp PID -o THREAD,tid,time 打印对应进程的线程列表，找到负载率最高线程的tid

  1. printf “%x\n” tid 将有问题线程ID转换为十六进制格式

  2. jstack PID | grep tid -A 30 打印线程的堆栈信息，此处tid为16进制形式。或者通过jstack pid > thread.txt命令，将栈信息打印到文件中，然后再打开文件，在文件中搜索tid的16进制值

cpu负载率高：

（1）首先使用top命令，查看当前cpu负载率，如下红框表示为15分钟、5分钟、1分钟的平均负载率

（2）使用vmstat命令查看进程、内存、I/O等的运行情况，vmstat m n表示每隔m秒进行一次采样，总共采样n次；vmstat m表示每隔m秒进行一次采样，会一直进行

各个字段含义如下：

字段名 含义
r 表示等待运行的进程数量
b 表示等待IO的进程数量（非中断睡眠状态）
swpd 表示使用虚拟内存大小
free 表示可用内存大小
buff 表示用作缓冲区的内存大小，缓冲区采用先进先出策略，如文件缓冲区用于暂存文件内容，之后会被同步到磁盘，可以实现批量写磁盘
cache 表示用作缓存的内存大小，缓存采用最近最少使用策略，如文件权限信息缓存，当读取时可以直接从缓存中读取；当不存在时，才会从磁盘中读取加载进缓存
si 表示每秒从虚拟内存（磁盘）写到内存的大小
so 表示每秒从内存写入虚拟内存（磁盘）的大小
bi 表示每秒读取磁盘的块数（每块1024字节）
bo 表示每秒写入磁盘的块数（每块1024字节）
in 表示每秒cpu产生中断数（包括时钟中断）
cs 表示每秒上下文切换次数，如用户态到内核态切换、线程的切换、进程上下文切换等
us 表示进程处于用户态运行的时间
sy 表示进程处于内核态运行的时间
id 表示空闲 CPU时间，id+us+sy=100
wa 表示io等待时间
st 表示CPU被其他虚拟机占用的时间

（3）使用pidstat命令获取切换次数最多的线程，pidstat -wt -n会每隔n秒显示所有进程和线程的上下文切换情况，pidstat -wt -p pid命令用于显示对应进程下每个线程的上下文切换次数

  • Cswch/s:每秒主动任务上下文切换数量，主动上下文切换指进程无法获取所需资源导致的上下文切换，如I/O、内存等系统资源不足
  • Nvcswch/s:每秒被动任务上下文切换数量，被动上下文切换指进程由于时间片已到等原因，被系统强制调度进而发生的上下文切换，如进程太多

（4）命令cat /proc/pid/status可以查看对应pid进程下的线程数

（5）printf “%x\n” tid 将有问题线程ID转换为十六进制格式

（6）jstack PID | grep tid -A 30 打印线程的堆栈信息，此处tid为16进制形式。或者通过jstack pid > thread.txt命令，将栈信息打印到文件中，然后再打开文件，在文件中搜索tid的16进制值

磁盘异常

磁盘满了解决过程

1、df -h 查找哪个磁盘使用率过高，可以找到对应的挂载目录
2、du -sh * 或 du -sh –max-depth=1 在对应挂载目录下查询文件占用的磁盘大小
3、echo “”>fileName 将磁盘占用较大的文件内容置为空
4、lsof | grep delete 查找那些已删除但空间未被释放的文件（注意可能需要切换用户）

磁盘读写异常解决过程

  1. iostat -d -k -x m n命令来每隔m小时输出磁盘I/O的详细情况，总共采样n次（k表示单位是kb/s，d表示输出结果不包含cpu信息，x表示输出I/O更详细统计信息）

名称 含义
rrqm/s 每秒对该设备的读请求被合并次数，文件系统会对读取同块(block)的请求进行合并
wrqm/s 每秒对该设备的写请求被合并次数，文件系统会对读取同块(block)的请求进行合并
r/s 每秒完成的读次数
w/s 每秒完成的写次数
rkB/s 每秒读数据量(kB为单位)
wkB/s 每秒写数据量(kB为单位)
avgrq-sz 平均每次IO操作的数据量(扇区数为单位)
avgqu-sz 平均等待处理的IO请求队列长度
await 平均每次IO请求等待时间(包括等待时间和处理时间，毫秒为单位)
svctm 平均每次IO请求的处理时间(毫秒为单位)
%util 采样周期内用于IO操作的时间比率，即IO队列非空的时间比率

dm-0表示centos逻辑卷的lvm设备
2. 通过iotop或可以查看哪些线程正在进行I/O读写，然后通过命令readlink -f /proc/*/task/tid/../..可以拿到此tid所在的进程pid
3. 若iotop命令不可用，也可以使用pidstat -d命令查看
4. 通过命令lsof -p pid可以查看此进程具体的读写情况
