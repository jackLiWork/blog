XSD介绍

2020-09-18 xml学习 xml, xsd 评论

###

1.简介

（1）XSD即XML结构定义, XML Schemas Definition。其本身就是用xml描述的, 且遵循xml语法规则。一份XML schema文件描述了XML文档的结构.

（2）基本规则：

  • 必须以 XML 声明开头
  • 必须拥有唯一的根元素
  • 标签必须与结束标签相匹配
  • 元素对大小写敏感
  • 所有的元素都必须关闭
  • 所有的元素都必须正确地嵌套
  • 必须对特殊字符使用实体

2、基本用法

（1）XML Schema 文件：

（2）XML 文档：

  • xmlns 表示note元素及其子元素定义所在的命名空间，
  • xsi:schemaLocation 表示note元素及其子元素定义的Schema 文件所在的位置
  • xmlns:xsi=”http://www.w3.org/2001/XMLSchema-instance"是因为我们使用了schemaLocation元素，其定义在http://www.w3.org/2001/XMLSchema-instance命名空间中

3、Schema 基本语法

（1）根元素： 元素是每一个 XML Schema 的根元素

  • xmlns:xs 规定schema中用到的元素和数据类型来自的命名空间，并且使用此命名空间中元素和数据类型应该使用前缀xs
  • targetNamespace 规定此schema中定义的元素所在的命名空间
  • xmlns 规定此schema中定义的元素所在的默认命名空间
  • elementFormDefault 指出任何 XML 实例文档所使用在此 schema 中声明过的元素必须被命名空间限定

（2）定义一个简单元素：指那些不会包含任何其他的元素或属性

（3）定义一个属性：可拥有指定的默认值或固定值。

（4）自定义简单类型(仅包含基本类型和约束，不包含元素)

（5）xs:restriction用于定义简单类型的约束

元素名 含义
enumeration 定义可接受值的一个列表
fractionDigits 定义所允许的最大的小数位数。必须大于等于0。
length 定义所允许的字符或者列表项目的精确数目。必须大于或等于0。
maxExclusive 定义数值的上限。所允许的值必须小于此值。
maxInclusive 定义数值的上限。所允许的值必须小于或等于此值。
maxLength 定义所允许的字符或者列表项目的最大数目。必须大于或等于0。
minExclusive 定义数值的下限。所允许的值必需大于此值。
minInclusive 定义数值的下限。所允许的值必需大于或等于此值。
minLength 定义所允许的字符或者列表项目的最小数目。必须大于或等于0。
pattern 定义可接受的字符的精确序列（可用正则表达式）。
totalDigits 定义所允许的阿拉伯数字的精确位数。必须大于0。
whiteSpace 定义空白字符（换行、回车、空格以及制表符）的处理方式。
collapse 换行、回车、空格以及制表符以及其他多个连续的空格会被替换为空格，开头和结尾的空格会被移除

（6）自定义复杂类型(包含其他元素)

（7）复合类型指示器：控制在文档中使用元素的方式

  • Order指示器：用于定义元素的顺序
  • Occurrence指示器：用于定义某个元素出现的频率
  • Group 指示器用于定义相关的数批元素。

（8） 元素：通过未被 schema 规定的元素来拓展 XML 文档
